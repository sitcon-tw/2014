「知行合一，學以致用」對求學階段的我們而言並不陌生； 然而跨出課業範疇，面對業界千奇百怪的問題與需求，又有多少人能真正發揮自己所學？ 學校傳授的知識，又有多少能夠實際應用於生活中呢？

由學生自發組成、自主籌備、以學生為核心，SITCON 學生計算機年會冀望展現學生最真實的熱情與能量， 使有成者能夠傳承己身經驗、初學者能對資訊領域有更深入的了解。 SITCON 2014，請與我們一同激盪創新的靈感、分享實作的經驗， 並與來自各地的學生共同探求理論與實務的平衡點。

台灣資訊工程教育的沉痾，創新，與改革
鄭永斌教授

國內資訊人才的培育，受限於僵化的聯考體制，導致許多進入相關科系的學生在就讀之後才發現興趣不符。

很不幸地，近十年的觀察，這樣子的狀況因為研究所入學的因素，除了造成研究所補習的盛行之外，因為實作會耗費大量的時間與精神，不利學業成績的維持。”考試引導教學”也許在大學課程中還不是問題，但是學生考量自身的切身利益，卻是集體地實踐與強化這條趨勢。台灣未來所繫的科技創新，軟體無疑將扮演主角，但是如果沒有紮實的實作能力當基礎，也終將只是南柯一夢。在這場演講中，我將整理我的觀察，並探討大環境的改革與創新。

程式人的寫作與出版 -- 關於「程式人雜誌」的一些經驗分享
陳鍾誠教授

一個大學教授，如果不寫論文，不申請國科會計畫，那他可以做甚麼呢？當然、這個世界不是只有論文與國科會計畫，可以做的事情還有很多！最重要的當然是教書，隨著時代演進，我從寫網誌、寫書、寫教材，逐漸開始進入程式類科普教育的領域。

2009 年，我出了第一本紙本書「系統程式」，然後、我開始想要自己做一台電腦，然後教學生怎麼自己做一台電腦，於是啟動了「開放電腦計畫」。後來、我發覺開放電腦計畫的產品，不應該是一台電腦，也不是一堆程式，而是一本教大家怎麼自己做電腦，包含所有軟硬體設計過程的書。

所以、我寫了「開放電腦計畫 -- 計算機軟硬體結構」這本書，而且一直到現在，都還沒有全部完成，或許、這本書永遠都不會完成也說不定⋯⋯接著、在 2012 年底，我們創辦了「程式人雜誌」，直到現在，每個月初一，我都會把雜誌放上 github，然後分享到「程式人雜誌」社團與我的 facebook 上，雖然我們的 facebook 社團已經有了 6000 人，但是、我並不知道這本雜誌真正的讀者有多少。今天、我想與大家分享的，就是在這些寫作與出版的過程當中，所得到的一些經驗⋯⋯

人機互動的方式隨著科技的進步不斷的改變，本小講討論了一個基於繪畫者想要畫出一朵「雲」的使用者需求，透過分析與文獻搜集彙集了解雲的呈現俱有以下特質「厚薄」、「托高」、「透明度」，在搜尋各種不同材質後，決定以長纖維棉花作為人機互動的界面，並且搭配光敏電阻的陣列借此感應人體與該朵棉花的互動情形，以Arduino接收 Serial訊號送至Processing進行繪圖的處理。本小講實作出一小型可謂療愈系的互動裝置，使用者可透過該界面完成一朵雲的創作，並且存取下來作為他用。在繪畫理論上基於漫畫式的畫雲原則，作為在Processing的雲朵實體呈現。

Auto Choose Connect With QR Code
SyneArt

QR Code 配對？在不需要親自詢問對方 IP、藍牙名稱的清況下要如何利用 QR Code 來把手機 to 電腦或是手機 to 手機配對這件事做簡化呢？

一般的配對方式，是以在 Client 端自行輸入 Server 端的 IP（有的還需自行輸入 port），在藍芽的部份則是手動選擇要連結的裝置名稱，接著再互相輸入配對碼後，手動進行連結。還有一種方式，是自動進行嘗試掃描對應，等到 Server 端有回應時，才進行連接，但這樣會進行長久的等待，而 Auto Choose Connect 改變了這些不方便的方式。
